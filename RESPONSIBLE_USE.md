# Responsible Use

This guide is intended for personal, educational, research, and authorized development use.

Please use it only with systems, bots, accounts, conversations, and infrastructure that you own or are explicitly authorized to modify.

## Expected practices

- Keep API keys, login cookies, tokens, environment variables, system prompts, and private conversation data out of public logs and screenshots.
- Treat model thinking output as potentially sensitive. Do not publish it automatically or expose it to users who did not opt in.
- Keep thinking separate from the final answer, history, memory, summaries, embeddings, analytics, and crash-recovery payloads unless there is a clear reason and explicit consent.
- Preserve a plain-text fallback so Telegram formatting failures cannot lose or duplicate the final answer.
- Verify the event structure against the installed Claude Code version before deployment.
- Respect platform terms, applicable law, privacy expectations, and the rights of other users.

## Uses this project is not intended to support

- Accessing or modifying systems without authorization
- Credential theft, secret extraction, or exposure of private prompts and conversations
- Surveillance, harassment, impersonation, or deceptive data collection
- Circumventing access controls or platform safeguards
- Packaging another person's work for sale without permission and attribution

This document describes the maintainer's intended and responsible use of the project. It does not replace or modify the repository license.
