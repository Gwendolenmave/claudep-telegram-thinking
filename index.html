旧调用

应用通常只读取终结对象的 .result，看不到中间 assistant content block。

新调用

  --output-format stream-json \
  --verbose

stdout 变成 NDJSON，每行一个 JSON event。实际事件形状应在本机版本中先验证。

详细复现步骤

1

能力探测

先在隔离环境运行，不要直接改 live bot。使用当前真实认证方式、模型和 wrapper 参数，只切换输出格式。

claude -p "A farmer has 17 sheep. All but 9 run away. How many remain?" \
  --output-format stream-json \
  --verbose \
  > /tmp/claude-stream.ndjson

jq -c 'select(.type=="assistant" or .type=="result")' \
  /tmp/claude-stream.ndjson

确认 assistant content 中出现独立 type:"thinking"，并确认最终正文仍来自终结 result.result。

2

逐行解析 NDJSON

thinking 只认 assistant.message.content[] 内的 thinking block；正文只认终结 result。不要把 stdout 所有行直接 join 成回复。

3

扩展现有返回类型

type ModelResult = {
  text: string;
  thinking?: {
    source: "reasoning";
    text: string;
  };
};

沿 provider → service → Telegram runtime 传递两个独立字段。

4

折叠渲染

使用 Telegram HTML <blockquote expandable>。thinking 只附在第一条消息，长消息优先截断 thinking，正文保持完整。

5

增加模式开关

/think auto/think off/think status

auto 只展示真实结构化 thinking；没有时静默只发正文。

6

隔离、测试、降级

thinking 不进入 history、memory、summary、embedding 或 crash-recovery 正文。HTML 失败时依次降级为 spoiler、普通引用、纯正文。

最小 TypeScript 解析器

export interface ParsedClaudeOutput {
  text: string;
  thinking?: { source: "reasoning"; text: string };
}

export function parseClaudeStreamJson(stdout: string): ParsedClaudeOutput {
  const thinkingParts: string[] = [];
  let finalText: string | undefined;

  for (const rawLine of stdout.split(/\r?\n/)) {
    const line = rawLine.trim();
    if (!line) continue;

    let event: unknown;
    try { event = JSON.parse(line); }
    catch { continue; }
    if (!isRecord(event)) continue;

    if (event.type === "assistant" && isRecord(event.message)) {
      const content = event.message.content;
      if (Array.isArray(content)) {
        for (const block of content) {
          if (isRecord(block) && block.type === "thinking" &&
              typeof block.thinking === "string" && block.thinking.trim()) {
            thinkingParts.push(block.thinking);
          }
        }
      }
    }

    if (event.type === "result" && typeof event.result === "string") {
      finalText = event.result;
    }
  }

  if (!finalText) throw new Error("No final result event received.");
  const thinking = thinkingParts.join("\n\n").trim();
  return { text: finalText, ...(thinking ? {
    thinking: { source: "reasoning", text: thinking }
  } : {}) };
}

function isRecord(value: unknown): value is Record<string, unknown> {
  return typeof value === "object" && value !== null;
}
正文权威源：终结 result.result。即使 assistant event 中也有 text block，也不要再拼一次，否则容易产生重复正文。

Telegram 折叠渲染

function escapeTelegramHtml(value: string): string {
  return value
    .replaceAll("&", "&")
    .replaceAll("<", "&lt;")
    .replaceAll(">", "&gt;")
    .replaceAll('"', "&quot;");
}

function renderMessage(text: string, thinking?: string): string {
  if (!thinking?.trim()) return escapeTelegramHtml(text);
  const clipped = thinking.length > 3500
    ? thinking.slice(0, 3500) + "\n\n… [thinking truncated]"
    : thinking;

  return [
    "<b>Thinking…</b>",
    `<blockquote expandable>${escapeTelegramHtml(clipped)}</blockquote>`,
    "",
    escapeTelegramHtml(text),
  ].join("\n");
}
网页显示说明：上方源码里的 &lt; 与 &gt; 是为了让示例在 HTML 页面中可见。复制后，转义函数应替换真实字符 < 与 >；发送请求使用 parse_mode="HTML"。

建议降级顺序：

expandable blockquote
→ spoiler
→ 普通 blockquote
→ 原始纯正文

每次失败后重试下一档，保证最终正文最多发送一次。
